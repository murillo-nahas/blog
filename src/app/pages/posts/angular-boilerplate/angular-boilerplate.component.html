Introdução

Boa Devs! Como vão? Espero que bem. Bom, nesse post de hoje, gostaria de apresentar para vocês um boilerplate de Angular, ou seja como eu faço as configurações iniciais para meus projetos em Angular e também darei algumas dicas. Sem enrolar muito, vamos começar.

Criação de Projeto

De início, criaremos um projeto normal utilizando o comando ng new *nome do projeto* e então aceitaremos o roteamento padrão do Angular e o template CSS. Depois disso, seu projeto deverá estar dessa forma:

Se está assim, então está tudo certo. Podemos começar entrando na pasta /src/app e começar a modificar alguns arquivos. Estando no diretório, iremos criar uma pasta chamada "pages", onde colocaremos nossa página home. É a partir desta página que nossa aplicação irá começar. Então, no terminal, certifique-se de estar dentro de /src/app e rode o comando do Angular para criação de componentes: ng generate component home. Isso deve gerar seu componente home e após isso seu projeto deve estar da forma abaixo, se estiver tudo certo!

Correção de arquivos e definição de rotas

Uma coisa que costumo fazer, como dito acima, é criar uma página home e defini-la como principal página do projeto, além disso, apagar todo o conteúdo de todos os arquivos iniciais. Portanto, vamos lá. Primeiro, iremos apagar todo o conteúdo dentro do arquivo app.component.html, na raiz do projeto, e então, definir uma rota padrão. No mesmo arquivo, preencha o HTML com: {{ defaultRoute }}

Agora, precisamos definir nossas rotas para que a aplicação entenda que a página home é a página principal e que sejamos direcionados para lá assim que a aplicação compila. Portanto, no arquivo app-routing.module.ts, iremos definir nossas rotas dentro do array de Rotas, nesse estilo:

Mas o que exatamente esse bloco de código faz? Vamos lá, irei te explicar.

Como vemos acima, desde o começo, temos os imports do Angular, que por sua vez importa o NgModule, para sinalizar ao Angular que nosso arquivo atual é um módulo. Abaixo, temos os módulos de rotas, que também são providos pelo próprio Angular e então, temos o import do nosso componente Home.

Então, temos nossa constante de rotas, que no caso passa um array de rotas. Exatamente nesse lugar é onde definimos as rotas de nossa aplicação. Veja que temos dois pedaços de código, que definimos as nossas rotas. São eles: {{ redirectRoute }} e {{ route }}. Este bloco de código define uma rota padrão home, que sempre que a URL da nossa aplicação cair em '', será redirecionado para o componente Home. Poderíamos definir outras rotas também, abaixo da última rota home definida. E todas seriam da mesma forma (exceto se estiver utilizando o lazy loading), mas irei deixar isso para outro post.

Considerações finais

Como vimos acima, criamos um projeto Angular com a definição de uma rota principal, que a partir dela, trabalharemos em nossa aplicação, definindo outras rotas, entre outras coisas. Temos outros assuntos que poderemos abordar futuramente, como a utilização de lazy loading, lint para formatação, entre outros assuntos.
